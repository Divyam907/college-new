Abstract

This report presents the design, development, and implementation of an Automated Attendance System Using Facial Recognition. The system replaces traditional manual attendance methods with a computer-vision-driven digital workflow that supports student enrollment through webcam capture, real-time identity verification using deep learning, subject-wise attendance marking, automated report generation in Excel format, and email-based communication to faculty and administrators.

The core recognition engine uses DeepFace with the VGG-Face model to generate 4096-dimensional facial embeddings for enrolled students. During attendance marking, faces detected in classroom images using MTCNN are compared against stored embeddings through Euclidean distance computation with a configurable threshold of 0.9. The system is built on Flask for the web portal, PostgreSQL for persistent data storage, and supports role-based access for administrators and college staff.

Beyond the base attendance functionality, the system introduces four advanced features currently under development: (a) Ambient Attention and Engagement Analytics using emotion and gaze detection to compute classroom engagement scores, (b) Zero-Click Continuous Attendance through CCTV/RTSP camera integration for fully automated frame capture, (c) Multi-Modal Anti-Spoofing with liveness detection to prevent proxy attendance through photograph-based fraud, and (d) Multi-Channel Notification via WhatsApp Business API for real-time alerts to parents, faculty, HODs, and the Dean.

The implemented system has been tested with a dataset of enrolled students and demonstrates successful face detection, identity matching, duplicate prevention, and report generation across daily, weekly, and monthly intervals.

Keywords: Facial Recognition, Attendance Automation, DeepFace, VGG-Face, Flask, PostgreSQL, MTCNN, Engagement Analytics, Liveness Detection, RTSP, WhatsApp API.

Chapter 1. Introduction

1.1 Background and Motivation

Attendance management is a fundamental administrative requirement in educational institutions. The conventional approach involves faculty manually calling names or circulating a paper sign-in sheet at the beginning of each lecture. While simple in concept, these methods introduce several problems when scaled across multiple subjects, sections, and semesters. Manual recording consumes valuable teaching time, introduces transcription errors during digitization, and creates opportunities for proxy attendance where one student signs on behalf of another [1], [2].

The growth of deep learning based computer vision, particularly in face recognition, has made automated identity verification a practical reality for controlled indoor environments such as classrooms [3], [4], [5]. Modern face recognition frameworks like DeepFace have abstracted the complexity of implementing deep neural network pipelines, making it possible for developers to integrate state-of-the-art recognition capabilities without training models from scratch [6], [11]. This technological maturity forms the primary motivation for the proposed system.

However, a realistic attendance system requires far more than a standalone face matching function. It demands integration with student enrollment workflows, relational database storage, subject management, user authentication, report generation, and communication channels. The motivation behind this project is to build a complete end-to-end platform that addresses all of these requirements in a single deployable application rather than limiting the scope to an isolated recognition experiment.

1.2 Problem Statement and Objectives

The central problem addressed by this project is the absence of an efficient, reliable, and traceable attendance mechanism for classroom environments. In a typical manual setting, faculty members spend several minutes verifying student presence and subsequently invest additional time preparing attendance summaries. This overhead grows proportionally with class size. Manual methods also offer limited protection against duplicate marking, record loss, and proxy attendance. Even when spreadsheets are used, the process remains dependent on manual data entry and repetitive administrative effort.

The objectives of the proposed system are as follows:

  1. To reduce the time spent during classroom attendance collection by automating identity verification through facial recognition.
  2. To use deep learning based facial embeddings (VGG-Face model via DeepFace) for verifying student identity from stored photo samples.
  3. To support role-based access separation between system administration and classroom operation through a web-based portal.
  4. To maintain persistent structured records in a PostgreSQL relational database with referential integrity constraints.
  5. To generate automated attendance reports in Excel format for daily, weekly, and monthly intervals.
  6. To deliver reports and alerts through multiple channels including email (SMTP) and WhatsApp (Business API).
  7. To integrate advanced features including engagement analytics, CCTV-based continuous attendance, and liveness-based anti-spoofing for a commercially viable product.

1.3 Benefits of Research

The proposed system offers the following benefits to educational institutions:

  1. Time Efficiency: Automated face recognition reduces per-class attendance time from 5–10 minutes of manual roll call to under 10 seconds of image capture and processing.
  2. Proxy Prevention: Unlike paper-based methods, the system verifies physical presence through biometric face matching, eliminating impersonation. The planned liveness detection module further prevents photograph-based spoofing attacks.
  3. Data Integrity: Attendance records are stored in a relational database with duplicate-prevention logic, ensuring that each student is marked at most once per subject per day.
  4. Automated Reporting: Faculty and administrators receive formatted Excel reports through automated email delivery, reducing manual report preparation effort.
  5. Scalability: The modular architecture allows the system to be extended to support multiple departments, buildings, or institutions by adding database partitions and camera nodes.
  6. Teaching Quality Insights: The proposed engagement analytics module provides educators with lecture-synced feedback on classroom attention levels, which is not available in any mainstream attendance product.
  7. Multi-Stakeholder Communication: WhatsApp integration ensures that parents, HODs, Deans, and faculty receive attendance information through a channel with significantly higher open rates than email.

1.4 Limitations of Research

The proposed system has the following limitations in its current stage:

  1. Lighting Dependency: Recognition accuracy is influenced by classroom lighting conditions. Poorly lit environments or strong backlighting can degrade face detection performance.
  2. Enrollment Quality: The recognition pipeline is dependent on the quality and variety of photos captured during student registration. If only one photo is captured at a narrow angle, the system may fail to match the student in classroom images captured from different perspectives.
  3. Threshold Sensitivity: The current distance threshold of 0.9 has been determined through iterative testing on the project dataset. It may require recalibration when deployed in different environments with different camera hardware.
  4. Hardcoded Class Count: The monthly attendance percentage calculation currently assumes a fixed total of 20 classes per month, which should ideally be derived from an actual timetable entity.
  5. Single Camera Input: The current web portal supports attendance marking through a single webcam or device camera. Multi-camera CCTV integration is under active development but not yet fully deployed.
  6. Advanced Features Under Development: The engagement analytics, liveness detection, CCTV integration, and WhatsApp notification modules are currently in the implementation phase and have not yet been tested at scale.

Chapter 2. Literature Survey

2.1 Introduction

This chapter reviews the existing literature on facial recognition technologies, attendance automation systems, and related deep learning architectures that form the theoretical foundation of the proposed system. The review covers foundational face recognition methods, modern deep learning approaches, face detection models, and relevant frameworks used in the implementation.

2.2 Literature Review

Turk and Pentland in [1] introduced the Eigenfaces method for face recognition in 1991, which uses Principal Component Analysis (PCA) to project face images into a low-dimensional eigenspace. This work established the viability of computational face recognition and served as a baseline for decades of subsequent research. However, Eigenfaces are sensitive to lighting, pose, and expression changes, which limits their effectiveness in unconstrained classroom environments.

Zhao et al. in [2] presented a comprehensive survey of face recognition techniques in 2003, covering holistic approaches (PCA, LDA), feature-based methods, and hybrid systems. The authors identified illumination variation, pose change, and occlusion as the three primary challenges for face recognition systems, all of which are relevant to classroom-based attendance marking.

Krizhevsky et al. in [3] demonstrated the power of deep convolutional neural networks (CNNs) with AlexNet, achieving breakthrough results on the ImageNet Large Scale Visual Recognition Challenge in 2012. This work catalyzed the adoption of deep learning for visual recognition tasks, including face recognition, by demonstrating that learned feature representations outperform handcrafted features.

Simonyan and Zisserman in [4] proposed VGGNet, which explored the impact of network depth on recognition accuracy. The VGG-Face model, derived from this architecture, uses a 16-layer deep CNN to produce 4096-dimensional face embeddings. This model is used as the primary embedding generator in the proposed system through the DeepFace framework.

Schroff et al. in [5] developed FaceNet at Google, which introduced the concept of learning a direct embedding mapping from face images to a compact Euclidean space where distances directly correspond to face similarity. FaceNet uses triplet loss training to achieve state-of-the-art face verification accuracy. The principle of embedding-based matching used in FaceNet directly informs the distance comparison strategy employed in the proposed system.

Parkhi et al. in [6] created VGG-Face, a deep face recognition model trained on a large-scale dataset of celebrity faces. The model generates fixed-length vector representations that can be compared using standard distance metrics. The proposed system uses VGG-Face as the default model for embedding generation.

Zhang et al. in [7] proposed MTCNN (Multi-Task Cascaded Convolutional Networks) for joint face detection and alignment. MTCNN uses a three-stage cascaded architecture (P-Net, R-Net, O-Net) that progressively refines face candidate regions while simultaneously predicting facial landmark locations. The proposed system uses MTCNN as the primary face detection backend for extracting facial regions from classroom images.

Deng et al. in [8] proposed RetinaFace, a single-shot multi-level face localization model that achieves robust face detection across different scales and poses. RetinaFace is available as an alternative detector backend in the proposed system through the DeepFace framework.

Deng et al. in [9] introduced ArcFace, which uses additive angular margin loss to learn highly discriminative face embeddings. ArcFace achieves state-of-the-art results on standard face verification benchmarks including LFW and MegaFace. This model represents a potential upgrade path for the embedding generator in future versions of the proposed system.

Taigman et al. in [10] developed DeepFace at Facebook, which achieved near-human-level face verification accuracy on the LFW benchmark. The approach uses a nine-layer deep neural network with 3D face alignment preprocessing. This work demonstrated the commercial viability of deep learning based face verification.

Serengil in [11] developed the DeepFace open-source framework (Python library), which provides a unified API for multiple face recognition backends including VGG-Face, FaceNet, ArcFace, and OpenFace. The proposed system uses this framework as the primary interface for embedding generation and face detection, significantly reducing implementation complexity.

Huang et al. in [12] created the Labeled Faces in the Wild (LFW) dataset, which has become the standard benchmark for evaluating face verification algorithms in unconstrained conditions.

He et al. in [13] introduced ResNet (Deep Residual Learning), which enabled training of very deep networks (up to 152 layers) through skip connections. ResNet architectures form the backbone of several modern face recognition models and represent a potential upgrade for the embedding generator.

Abadi et al. in [14] presented TensorFlow, the machine learning framework used as the backend for DeepFace and Keras in the proposed system. TensorFlow provides efficient GPU-accelerated tensor operations for neural network inference during embedding generation.

Viola and Jones in [24] proposed a rapid object detection framework using Haar-like features and a cascaded AdaBoost classifier, which became the dominant real-time face detector for over a decade before deep learning methods overtook it. Their work established the principle of cascaded rejection that is refined in modern detectors like MTCNN.

Goodfellow et al. in [25] authored a comprehensive textbook on deep learning, covering convolutional networks, representation learning, and generative models. The theoretical foundations described in this text underpin the feature extraction and embedding generation process used in the proposed system.

2.3 Inferences Drawn from Literature Review

The following inferences are drawn from the literature review:

  1. Deep learning based face recognition models (VGG-Face, FaceNet, ArcFace) significantly outperform classical methods (Eigenfaces, Fisherfaces) in terms of accuracy and robustness to environmental variation [1], [5], [9].
  2. Embedding-based approaches, where faces are mapped to fixed-length vectors in Euclidean space, provide an efficient and scalable mechanism for identity comparison without retraining the model for each new identity [5], [6].
  3. MTCNN provides robust multi-face detection and alignment suitable for classroom group photographs with moderate occlusion [7].
  4. The DeepFace framework significantly reduces implementation complexity by providing a unified API across multiple recognition and detection backends [11].
  5. No existing literature presents a complete attendance system that integrates face recognition with engagement analytics, multi-channel alerting (WhatsApp), and liveness detection in a single deployable platform.
  6. The majority of existing attendance automation studies limit their scope to face detection and recognition, without addressing the administrative pipeline of report generation, role-based access, and multi-stakeholder communication.

Chapter 3. Proposed Work

3.1 Introduction

This chapter presents the proposed system architecture, the core attendance module, and the four advanced features that differentiate the system from existing market solutions. The proposed work encompasses both the currently implemented modules and the features under active development. The system follows a modular architecture in which enrollment, recognition, record storage, reporting, and communication are handled by separate Python modules, enabling independent development and testing of each component.

3.2 Proposed System Architecture

The system architecture consists of five primary layers: the presentation layer (Flask web portal), the application logic layer (Python modules), the recognition engine layer (DeepFace with VGG-Face), the data storage layer (PostgreSQL), and the communication layer (SMTP email and WhatsApp API). These layers interact through well-defined interfaces to support two distinct operating modes: interactive web-based attendance marking and scheduled batch processing.

Fig. 3.1 Proposed System Architecture

As illustrated in Fig. 3.1, the system begins with image acquisition through a webcam or IP camera. The captured image passes through the face detection module (MTCNN), which extracts individual facial regions. Each facial region is converted into a 4096-dimensional embedding vector by the VGG-Face model. These embeddings are compared against the stored embedding repository using Euclidean distance computation. Matches that satisfy the threshold constraint (distance ≤ 0.9) are accepted, and the corresponding attendance records are inserted into the PostgreSQL database with duplicate prevention.

3.3 Core Attendance Module

The core attendance module supports two operational modes. In the first mode (interactive portal), an administrator registers students through a webcam-enabled interface, stores images in the dataset directory, and regenerates face embeddings. A college staff user then captures a classroom image, selects the active subject, and the system detects all faces, compares embeddings, and marks attendance for recognized students in real time. In the second mode (scheduled batch), images placed in the images directory with filenames in HH-MM-SS.jpg format are automatically processed at configured times, with subject mapping derived from the image timestamp and the subject time-range table.

3.3.1 Admin Portal

The admin portal provides a dashboard displaying summary statistics (total students, today's attendance count, number of subjects), student registration with webcam photo capture supporting up to 5 photos per student, student management with deletion capability, and a report dispatch interface supporting daily, weekly, and monthly report types with date selection and recipient preview.

3.3.2 College Attendance Portal

The college portal provides a subject selection interface with radio buttons displaying subject names and time slots, live camera feed for classroom image capture, real-time processing feedback showing identified students with confidence scores, and an attendance records viewer with 30-day history filtering.

3.3.3 Authentication and Role Management

Role separation is handled through the users table and Flask session-based authentication. Passwords are stored using Werkzeug's hashing functions rather than plain text. Three custom decorators (@login_required, @admin_required, @college_required) enforce access restrictions at the route level.

3.4 Advanced Feature: Ambient Attention and Engagement Analytics

This feature transforms the system from a pure attendance tool into a teaching quality improvement platform. While marking attendance, the system uses emotion detection and gaze estimation to compute a per-lecture Engagement Score.

3.4.1 Technical Approach

The DeepFace framework includes a built-in emotion analysis module that classifies facial expressions into seven categories: angry, disgust, fear, happy, sad, surprise, and neutral. During attendance processing, after face detection, each detected face is additionally analyzed for emotional state. Gaze direction estimation is performed using facial landmark positions (eye corners, pupil centers) extracted during MTCNN alignment. A student is classified as "engaged" if the gaze vector points toward the front of the classroom (within a configurable angular tolerance) and the emotional state is neutral, happy, or surprise (indicating active cognitive processing).

3.4.2 Engagement Score Calculation

The Engagement Score for a lecture is computed as follows:

Engagement Score = (Number of Engaged Students / Total Detected Students) × 100 (3.1)

The system generates a time-series graph of the Engagement Score sampled at multiple points during the lecture (e.g., every 15 minutes when using CCTV integration). Faculty receive a post-lecture report indicating which segments of the lecture had highest and lowest engagement, enabling targeted improvement of teaching methods.

3.5 Advanced Feature: Zero-Click Continuous Attendance (CCTV/RTSP Integration)

Most existing attendance systems require active teacher participation — capturing a photo, pressing a button, or scanning each student individually. The proposed CCTV/RTSP integration module eliminates all manual intervention by connecting directly to existing IP cameras in classrooms.

3.5.1 Technical Approach

The module uses OpenCV's VideoCapture with RTSP stream URLs to connect to IP cameras. The system captures frames at configurable random intervals (default: every 15 minutes) throughout the lecture duration. Each captured frame is processed through the standard attendance pipeline (face detection, embedding, matching, insertion). The use of multiple captures per lecture serves two purposes: (a) it increases the probability of detecting students who may be temporarily occluded or turned away in a single capture, and (b) it prevents partial bunking where a student marks attendance and then leaves the class.

3.5.2 Data Flow

The RTSP module operates as a background thread that reads the subject timetable from the database, determines which cameras correspond to which classrooms, and activates capture only during scheduled lecture windows. Captured frames are stored temporarily with timestamps and processed asynchronously to avoid blocking the camera feed.

3.6 Advanced Feature: Multi-Modal Anti-Spoofing (Liveness Detection)

A critical vulnerability in photograph-based face recognition systems is susceptibility to spoofing attacks, where an adversary presents a printed photograph or a digital image on a smartphone screen in front of the camera to impersonate another student. The proposed liveness detection module addresses this vulnerability.

3.6.1 Technical Approach

The anti-spoofing module implements a multi-layer detection strategy:

  1. Texture Analysis: Live faces exhibit micro-textures (pores, fine wrinkles) that are absent in printed or screen-displayed photographs. A Local Binary Pattern (LBP) texture descriptor is extracted from the detected facial region and classified as live or spoof using a trained binary classifier.
  2. Screen Glare Detection: When a face is displayed on a smartphone or laptop screen, the camera captures screen-specific artifacts including periodic brightness patterns (flicker) and surface reflections. The module analyzes the high-frequency components of the facial region for these artifacts.
  3. Micro-Expression Verification: For individual attendance scenarios (not group photos), the system can request a micro-expression challenge such as a blink or a head turn, verified through consecutive frame analysis.

4.2 Implementation Strategy

4.2.1 Facial Embedding Generation Algorithm

The embedding generation process, implemented in gen_embed.py, traverses the dataset directory where each subdirectory represents a registered student containing one or more facial images. For each image, the VGG-Face model extracts a 4096-dimensional feature vector. All embeddings and corresponding student names are stored in embeddings.csv for efficient retrieval during attendance marking.

Algorithm 1: Embedding Generation
Input: Dataset directory D containing student subdirectories
Output: embeddings.csv file

1.  Initialize empty lists: embeddings[], names[]
2.  FOR each subdirectory student_folder IN D:
3.      FOR each image_file IN student_folder:
4.          TRY:
5.              embedding ← DeepFace.represent(image_file, model='VGG-Face')
6.              Append embedding to embeddings[]
7.              Append student_folder.name to names[]
8.          CATCH Exception:
9.              Log error and continue
10. Create DataFrame from embeddings[] and names[]
11. Save DataFrame to embeddings.csv

4.2.2 Attendance Marking Algorithm

The attendance inference pipeline, implemented in Attendance_update_db.py, processes a classroom image to detect multiple faces and match each against the stored embedding repository.

Algorithm 2: Attendance Marking from Group Image
Input: Classroom image I, Subject ID S, Date D
Output: List of recognized students marked present

1.  faces[] ← DeepFace.extract_faces(I, detector='mtcnn')
2.  stored_embeddings ← Load from embeddings.csv
3.  FOR each face F IN faces[]:
4.      Crop face region from image I
5.      query_embedding ← DeepFace.represent(F, model='VGG-Face')
6.      distances[] ← EuclideanDistance(query_embedding, stored_embeddings)
7.      min_distance ← MIN(distances[])
8.      best_match_index ← ARGMIN(distances[])
9.      IF min_distance ≤ 0.9 THEN:
10.         student_name ← names[best_match_index]
11.         student_id ← LookupStudentID(student_name)
12.         IF NOT DuplicateExists(D, S, student_id) THEN:
13.             INSERT INTO attendance(date, subject_id, student_id, image)
14.             Add student_name to recognized_list
15.     ELSE:
16.         Mark face as "Unknown"
17. RETURN recognized_list

The Euclidean distance between a query embedding eq and a stored embedding ei is computed as:

di = ‖eq − ei‖2 = √(Σ(eq,j − ei,j)²)   for j = 1 to 4096 (4.1)

The confidence score is derived as:

c = 1 / (1 + dmin) (4.2)

4.2.3 Report Generation Pipeline

The report generation module (generate_report.py) supports three report types, each implemented as a separate function that queries the PostgreSQL database using pandas:

  • Daily Report: Pivots attendance by subject and student name for a selected date.
  • Weekly Report: Aggregates the count of days present per student per subject between a start and end date.
  • Monthly Report: Computes total classes (currently hardcoded at 20), present count, absent count, and attendance percentage for each student-subject pair for a given month.

Generated reports are exported as Excel files (.xlsx) using openpyxl and dispatched via SMTP email to configured faculty and director recipients through send_report.py.

Chapter 6. Conclusion and Future Scope

6.1 Conclusion

This report presented the design, development, and implementation of an Automated Attendance System Using Facial Recognition. The system combines deep learning based face recognition using DeepFace with the VGG-Face model, webcam-based student enrollment through an admin portal, role-based web access through Flask, PostgreSQL-backed persistent data management, automated Excel report generation for daily, weekly, and monthly intervals, and SMTP email dispatch into a single integrated platform.

The implemented system successfully demonstrates the following capabilities:

  1. Student registration through a webcam-enabled interface with automatic embedding generation and database insertion.
  2. Multi-face attendance marking from a single classroom image using MTCNN face detection and VGG-Face embedding comparison with a threshold of 0.9.
  3. Duplicate prevention logic that ensures each student is marked at most once per subject per day.
  4. Subject-wise attendance marking through both scheduled batch processing (time-based) and interactive portal-based operation (manual subject selection).
  5. Automated report generation in Excel format with email delivery to configured recipients.
  6. Role-based access control separating administrator functions (enrollment, reporting) from college staff functions (attendance marking, record viewing).

The system addresses the core problem of manual attendance inefficiency by reducing the per-class attendance process from several minutes of roll call to under 10 seconds of image capture and automated processing. The modular architecture ensures that individual components can be updated, replaced, or extended without disrupting the overall workflow.

6.2 Future Scope

The proposed system has significant scope for enhancement in the following directions, several of which are currently under active development:

  1. Engagement Analytics (Under Development): Integration of emotion detection and gaze estimation to compute per-lecture engagement scores, providing faculty with actionable feedback on teaching effectiveness.
  2. CCTV/RTSP Integration (Under Development): Direct connection to existing IP cameras for zero-click continuous attendance with multiple captures per lecture to prevent partial bunking.
  3. Liveness Detection (Under Development): Multi-modal anti-spoofing using texture analysis, screen glare detection, and micro-expression verification to guarantee proxy-proof attendance.
  4. WhatsApp Multi-Channel Alerts (Under Development): Integration with WhatsApp Business API through Twilio for instant notifications to parents, faculty, HODs, and the Dean with configurable notification preferences.
  5. Timetable Entity: A dedicated timetable database table to replace the hardcoded monthly class count assumption in attendance percentage calculations.
  6. Alternative Embedding Models: Evaluation of ArcFace [9] or FaceNet [5] as potential replacements for VGG-Face to improve matching accuracy under challenging conditions.
  7. Dashboard Analytics: Addition of graphical dashboards showing attendance trends, low-attendance risk students, and subject-wise patterns over time.
  8. Multi-Department Support: Extension of the database schema to support multiple departments, sections, and semesters within a single deployment.
  9. Mobile Application: Development of a companion mobile application for attendance marking on smartphones using the device camera.
  10. Formal Benchmark Evaluation: Rigorous evaluation of recognition accuracy using standard datasets (LFW [12]) with precision, recall, and F1-score metrics.
